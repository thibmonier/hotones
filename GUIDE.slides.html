What is HotOnes?

HotOnes is a project profitability management system for web agencies.

It helps agencies understand and optimize project profitability by tracking the full lifecycle:

The Most Important Concept

üè¢ Multi-Tenancy: Company-Based Isolation

Every piece of data belongs to a Company.

This is a multi-tenant SaaS platform where:

  • Company entity is the root tenant
  • CompanyOwnedInterface marks tenant-scoped entities
  • Automatic filtering prevents cross-tenant data leakage
  • Security enforced via CompanyVoter

Why Multi-Tenancy Matters

üîí Critical Security Rule

When adding new features, any entity that stores customer data MUST:

  • Implement CompanyOwnedInterface
  • Have a company relationship
  • Be filtered in repositories

This is not optional - it prevents data leakage between tenants.

üîí Security & Authorization

Understanding the security model

Role Hierarchy

ROLE_USER (base - all authenticated)
  ‚îî‚îÄ‚îÄ ROLE_INTERVENANT (contributor)
      ‚îî‚îÄ‚îÄ ROLE_CHEF_PROJET (project manager)
          ‚îî‚îÄ‚îÄ ROLE_MANAGER (team manager)
              ‚îî‚îÄ‚îÄ ROLE_COMPTA (accounting)
                  ‚îî‚îÄ‚îÄ ROLE_SUPERADMIN (full admin)

Security Patterns

  • CSRF Protection: Enabled by default on all forms
  • Controller Security: #[IsGranted('ROLE_X')] attributes
  • API Security: Expressions in #[ApiResource]
  • Multi-Tenant: CompanyVoter enforces data isolation
  • 2FA: Available but must be manually enabled per user

2FA Setup: /me/2fa/enable

Company-Based Security

The CompanyVoter ensures users can only access their company's data.

  • Automatic filtering in repositories
  • Controllers inject CompanyContext
  • Forms automatically assign company
  • Prevents cross-tenant data leakage

Critical: This is the primary security mechanism for multi-tenancy. Do not bypass it.

Multi-Tenant Development

Always remember: this is multi-tenant

  • ‚úÖ DO: Implement CompanyOwnedInterface for user data
  • ‚úÖ DO: Filter by company in repositories
  • ‚úÖ DO: Inject CompanyContext in controllers
  • ‚ùå DON'T: Query data without company filtering

Problems you might encounter

Port Conflicts

Problem: "Port already in use" errors

Affected ports:

  • 8080 - Web server (nginx)
  • 3307 - MariaDB

Asset Build Issues

Problem: CSS/JS changes not appearing

Solutions:

  • Check if assets were built: ls public/build/
  • Rebuild: ./build-assets.sh dev
  • Use watch mode: ./build-assets.sh watch
  • Clear browser cache

Test Environment

Problem: Tests using wrong database

Tests use SQLite, not MariaDB

  • Configured in .env.test
  • Provides isolation without external DB
  • Subtle differences from MariaDB possible
  • If tests fail but app works, check SQL syntax differences

PHP Version Issues

Problem: Syntax errors or unexpected behavior

This project requires PHP 8.5+

  • Uses PHP 8.4 property hooks
  • Cutting-edge syntax
  • Ensure Docker image matches: php:8.5.1-fpm
  • IDE/tooling must support PHP 8.4+

2FA Lockout

Problem: Can't login after enabling 2FA

Solution: Disable 2FA via database
