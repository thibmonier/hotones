## 1. Executive Summary `INFERRED`

> INFERRED — generated by analyzing `README.md`, `composer.json`, role hierarchy, controllers and 8 Symfony domain events. Human validation required.

**HotOnes** is a multi-tenant SaaS platform that manages the **profitability of web/digital agencies**. It consolidates four data streams to compute margin in near-real time:

1. **Sales & quotes** — daily rate (TJM), quote lifecycle, won/lost ratio.
2. **Capacity & timesheets** — staffing, planned days, actual time logged, overload alerts.
3. **Costs** — internal cost rate (CJM), purchases, expenses, employment periods.
4. **Operational KPIs** — project health, margin, budget consumption, payment due, NPS, contributor satisfaction.

Each tenant (`Company`) hosts multiple business units, contributors, clients, projects, orders, invoices and timesheets, isolated end-to-end.

The platform exposes:
- A web app (Twig + Symfony UX + Stimulus + Bootstrap 5 / theme Skote) for daily users.
- An admin backoffice on EasyAdmin (`/backoffice`).
- A REST API on API Platform 4.2 with JWT auth (`/api`, OpenAPI `/api/docs`).
- AI-assisted features (chatbot, predictions, forecasting) backed by Anthropic Claude / OpenAI / Gemini through `symfony/ai-bundle`.
- Operational integrations with HubSpot (CRM) and BoondManager (staffing).

**Stack**: Symfony 8.0.x / PHP 8.5 / MariaDB 11.4 / Redis 7 / Docker / Sentry / SonarCloud / Snyk. ~63 entities, ~80 controllers (~338 routes), ~48 services, ~46 CLI commands, ~102 test files.

## 2. Problem Statement `INFERRED`

> INFERRED from domain model (`Order`, `Project`, `Timesheet`, `Invoice`, `FactForecast`, `ContributorSatisfaction`, `ProjectHealthScore`) and 11 `NotificationType` enum cases reflecting alert categories. Human validation required.

Web / digital service agencies suffer from fragmented operational data:
- Quotes live in CRM, time in spreadsheets, billing in accounting tools, capacity in calendars.
- Margin is recomputed monthly at best, often per-project after the fact.
- Project drift (budget overrun, low margin, contributor overload) is detected too late.
- Multi-company groups cannot consolidate KPIs without manual export/import.

**HotOnes** unifies the operational chain (CRM → quote → project → staffing → timesheet → invoice → cash-in) under one tenant boundary, computes margin, raises alerts when thresholds are crossed (`PROJECT_BUDGET_ALERT`, `LOW_MARGIN_ALERT`, `CONTRIBUTOR_OVERLOAD_ALERT`, `PAYMENT_DUE_ALERT`, `KPI_THRESHOLD_EXCEEDED`, `TIMESHEET_PENDING_VALIDATION`, `TIMESHEET_MISSING_WEEKLY`), and surfaces them through a notification system + dashboards (Sales, HR, Staffing, Treasury, Project health).

**⚠️ To validate**: business pain hierarchy, target ICP (agency size, geography), competing tools currently displaced, willingness-to-pay drivers.
